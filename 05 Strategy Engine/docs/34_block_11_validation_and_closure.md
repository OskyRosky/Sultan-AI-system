## Validation Intent

Validation proves only that a closed conceptual package can be packaged into an immutable audit-ready dossier.

It does not execute handoff, authorize backtesting, validate performance, confirm edge, or authorize trading.

## Expected Test Coverage

The Block 11 tests must show that:

- a valid dossier can be created from a valid closure record;
- non-closure origins are rejected;
- all required dossier sections are required;
- duplicate section types are rejected;
- manual traceability mismatches are rejected;
- unsupported handoff status is rejected;
- downstream questions, pending requirements, non-approval statement, and audit reference are enforced;
- closure traceability and upstream falsification references are preserved;
- dossiers are immutable;
- dossiers do not expose operational handoff, backtest, PnL, performance, execution, or trading fields.

## Closure Conditions

Block 11 is complete when:

- the Strategy Dossier & Handoff contract is documented;
- implementation is covered by tests;
- dossiers are constrained to valid closure records;
- handoff status remains `dossier_prepared_pending_final_audit`;
- no real candidates are packaged from real data;
- no operational handoff, backtesting, PnL, performance metrics, execution, or data integration are created;
- 04 Research Layer is not modified.
