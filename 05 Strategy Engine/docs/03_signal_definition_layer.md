# 03 Signal Definition Layer

## Purpose

The Signal Definition Layer describes abstract strategic signals. A signal is a design concept that can later be tested by downstream systems.

## Signal Boundaries

Signals in 05 are abstract. They may describe direction, condition, context, threshold intent, or hypothesis linkage, but they are not executable instructions.

## Explicit Non-Scope

The Signal Definition Layer does not create:

- Orders.
- Trades.
- Execution instructions.
- Position sizes.
- Live alerts.
- Backtest-ready alpha code.

## Governance Requirement

Every future signal must be traceable to governed 04 Research Layer artifacts and must carry assumptions, limitations, and expected falsification framing.
